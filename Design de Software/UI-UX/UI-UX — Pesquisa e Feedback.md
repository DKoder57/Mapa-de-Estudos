# Pesquisa e Feedback no Lean UX

[[UI-UX]] → Módulo 2

## Pesquisa Contínua e Colaborativa

No Lean UX, a pesquisa deve ser **contínua e colaborativa**, substituindo processos tradicionais com início e fim. Ocorre durante cada sprint, com pequenas atividades de levantamento e análise de dados.

### Collaborative Discovery

Abordagem que envolve toda a equipe para aprender com os clientes e testar hipóteses:

1. Revisar perguntas
2. Criar guia de entrevistas
3. Dividir a equipe em pares para realizar entrevistas
4. Registrar feedback
5. Ao final, pedir sugestões para outros participantes

---

## Modelo Três-Doze-Um

Modelo sugerido para aprendizagem contínua:

> Testar **3 usuários** até as **12 horas**, **1 vez por semana**

Estruturado em: planejamento → refinamento do MVP → testes → revisão dos resultados

```
Segunda: Iniciar recrutamento + decidir o que será testado
Terça:   Refinar o que será testado
Quarta:  Refinar + escrever script do teste + finalizar recrutamento
Quinta:  Dia de testes + revisar achados com a equipe
Sexta:   Planejar próximos passos com base nos achados
```

---

## Recrutamento de Usuários

Pode ser feito internamente ou terceirizado, com compensação para participantes. Após os testes, a equipe deve:

- Transcrever notas em cartões ou post-its
- Organizar em temas
- Construir compreensão compartilhada

---

## Práticas para Maximizar o Aprendizado

1. **Procurar padrões** — explorar opiniões comuns; manter outliers separados para análise futura
2. **Validar o feedback** — confrontar com dados de outras fontes (e.g., e-mails de suporte)
3. **Identificar padrões ao longo do tempo** — pesquisa contínua revela padrões em questões menos urgentes
4. **Testar o que está pronto** — esboços, wireframes, protótipos — com expectativa clara sobre o tipo de feedback

---

## Artefatos de Design e seus Feedbacks

|Artefato|Propósito do feedback|
|---|---|
|**Esboços**|Validar conceitos e facilitar conversas durante entrevistas|
|**Wireframes**|Avaliar hierarquia de informação, taxonomia, navegação e arquitetura da informação|
|**Protótipo alta fidelidade (não clicável)**|Feedback detalhado sobre estética, paleta de cores e clareza das CTAs|
|**Protótipos clicáveis**|Simulações mais interativas|
|**Protótipos codificados**|Maior fidelidade funcional; testes com dados reais e integração com sistemas|

## Métodos Qualitativos vs Quantitativos

> [!NOTE] Diagrama de Rohrer (2012) Organiza técnicas em quadrantes por:
> 
> - Abordagem: qualitativa (não estruturada / amostra não probabilística) vs quantitativa (estruturada / amostra probabilística)
> - Envolvimento: atitudinal vs comportamental
> 
> **Exploratórios (qualitativos):** entrevista de profundidade, diário de uso, design participativo, grupo focal, estudos etnográficos **Exploratórios (comportamentais):** teste de usabilidade, eye tracking, benchmarking, mapas de calor **Conclusivos (qualitativos):** teste de desabilidade, teste de conceito, avaliação heurística, card sorting **Conclusivos (quantitativos):** survey, large scale unmoderated (teste A/B), estatísticas de uso (analytics), monitoramento de cliques de funil

## No Projeto Landing Page

O MVP foi testado com 10 usuários. Principais achados:

- **Clareza do CTA:** 8/10 usuários entenderam a ação imediatamente
- **Exemplos visuais:** sugestão de inserir demonstrações rápidas das funcionalidades
- **Confiança:** adicionar mais provas sociais (casos de sucesso e estatísticas)
- **Formulário:** explicar o benefício do preenchimento para aumentar confiança
- **CTA:** texto mais chamativo e claro
- **Versão demo:** oferecer período gratuito de 7 dias com todos os recursos
