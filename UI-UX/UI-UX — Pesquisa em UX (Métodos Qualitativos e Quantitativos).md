# Pesquisa em UX — Métodos Qualitativos e Quantitativos

[[UI-UX]] → Módulo 3 
## Choosing the Right Method

A escolha entre qualitativo, quantitativo ou combinação depende do objetivo:

|Método|Foco|Quando usar|
|---|---|---|
|**Qualitativo**|Experiências, opiniões, motivações|Entender o "porquê" por trás de comportamentos|
|**Quantitativo**|Dados numéricos e mensuráveis|Identificar padrões, medir desempenho em escala|
|**Combinado**|Ambos|Visão mais completa — insights detalhados + impacto mensurável|

---

## Técnicas Qualitativas

Perguntas abertas que permitem ao usuário descrever suas experiências. Exploram aspectos subjetivos.

**Exemplos de perguntas:**

- "O que você achou do processo de navegação?"
- "Como você se sentiu ao realizar esta tarefa?"
- "Quais foram os principais pontos positivos e negativos de aplicações similares?"
- "Qual é o principal objetivo que você espera alcançar com este aplicativo?"

---

## Técnicas Quantitativas

Focam em dados numéricos e objetivos. Medem preferências e comportamentos em escala.

**Exemplos de perguntas:**

- "Em uma escala de 1 a 5, como você avalia a facilidade de navegação?"
- "Com que frequência você usaria este app?" (diariamente/semanalmente/mensalmente/raramente)
- "Quais funcionalidades você considera essenciais?" (seleção múltipla)
- "O app atendeu suas expectativas?" (Sim/Não)

---

## Google Forms para Coleta de Dados

Ferramenta acessível para coleta de dados qualitativos e quantitativos. Permite:

- Múltipla escolha, questões abertas, escalas, upload de arquivos
- Exportação direta para Google Sheets / Excel
- Integração com Power BI para análise avançada

> [!TIP] Boas práticas no formulário
> 
> - Perguntas curtas e diretas
> - Combinar fechadas (quantitativas) com abertas (qualitativas)
> - Nem todas as perguntas precisam ser obrigatórias
> - Manter anônimo quando possível para aumentar participação
> - Uma pergunta por vez aumenta a taxa de resposta

**Exemplos de perguntas para pesquisa de UX:**

```
A) "Qual é o principal objetivo com este app?" (aberta)
B) "Com que frequência usaria?" (múltipla escolha)
C) "Quais funcionalidades são essenciais?" (seleção múltipla)
D) "Já usou algo semelhante? Pontos positivos e negativos?" (aberta)
E) "Nível de dificuldade de navegação?" (escala 1–5)
F) "O app atendeu às suas expectativas?" (Sim/Não)
```

## Ferramentas Especializadas em UX

|Ferramenta|Capacidade|
|---|---|
|**UserTesting**|Usuários reais testam a aplicação; relatórios detalhados sobre a experiência|
|**Hotjar**|Mapas de calor, gravações de sessão — onde usuários clicam e interagem|
|**Optimal Workshop**|Card sorting, testes de usabilidade — organização e hierarquia de informação|
|**Maze**|Rastreia comportamento em protótipos; testes A/B, score card|
|**Lookback.io**|Gravações + entrevistas ao vivo em tempo real|

---

## Organização e Análise dos Dados

1. **Exportar** respostas do Google Forms para Google Sheets ou Excel
2. **Limpar** dados inconsistentes ou incompletos
3. **Integrar** ao Power BI para dashboards visuais
4. **Interpretar** padrões: gráficos de barras, dispersão, mapas de calor
5. **Para dados qualitativos:** análise temática — categorizar palavras-chave recorrentes em temas (ex.: "facilidade de uso", "suporte", "dificuldade de navegação")

> [!NOTE] Análise temática Revisar respostas abertas → identificar palavras-chave → organizar em categorias → destacar pontos críticos prioritários.
